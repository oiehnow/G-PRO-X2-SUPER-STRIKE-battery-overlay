로지텍 pro x2 superstrike 를 사용중인데 마우스의 배터리 잔량과 사용할수 있는 평균 시간을 나타내는 어플이 이미 로지텍에서 지원하는 g허브가 있긴해 하지만 난 따로 오버레이로 작게 껏다 켰다 할수있는 기능으로 배터리 잔량과 남은 평균시간을 띄웠으면 좋겠어 혹시 코딩으로 풀어낼 방법이 있을까?

마우스 모델명을 먼저 확인하고, 배터리 잔량을 코드로 읽어오는 방법이 실제로 가능한지 검색해볼게.마우스는 확실해. LIGHTSPEED 무선에 60~90시간 배터리짜리고, 흥미롭게도 이 모델은 웹 기반 G HUB 드라이버를 쓴다는 얘기가 있네. 배터리를 코드로 읽어오는 부분이 핵심이라 그쪽을 더 파볼게.PRO X2가 신형이라 한 가지 중요한 함정이 보여 — 표준 HID++가 아니라 새로 리버스엔지니어링된 "Centurion" 프로토콜을 쓴대. 이게 통합 방법을 결정하니까 LGSTrayEx 쪽을 직접 확인해볼게.마우스 확인했고, 이게 코딩으로 풀 수 있는지도 검증됐어. 결론부터: 가능해. 다만 네 모델 때문에 함정이 하나 있어서 그 얘기부터 할게.

먼저 메모리에 이 프로젝트 맥락을 짧게 남겨둘게.이제 본론. 결론부터 말하면 **충분히 코딩으로 풀 수 있어.** 근데 네 마우스가 신상이라 작전 짜기 전에 알아둬야 할 함정이 하나 있어.

**함정: PRO X2 Superstrike는 "표준 무기"가 아니야**

보통 로지텍 무선 마우스는 HID++ 2.0이라는 공통 프로토콜로 배터리를 보고해. 인터넷에 굴러다니는 파이썬 스니펫 갖다 쓰면 끝나는 경우가 많지. 그런데 Superstrike를 포함한 PRO X 2 / G522 같은 최신 라인은 **Centurion이라는 새 프로토콜**을 써. G HUB도 이 모델은 아예 웹 기반 드라이버로 갔을 정도로 구조가 다르고, 이건 커뮤니티에서 따로 리버스엔지니어링해야 했던 물건이야. 즉, 너 혼자 hidapi로 raw HID 패킷 직접 까서 구현하는 건 가능은 하지만 진흙탕 백병전이야. 굳이 거기서 피 흘릴 필요 없어.

**작전: 정찰병은 외주, 너는 화력만 얹는다**

`LGSTrayEx`라는 오픈소스 트레이앱(C#, GPL-3.0, 윈도우 전용)이 이미 그 Centurion 프로토콜을 깨놨고, 호환 목록에 `(Lightspeed) G Pro X 2 Mouse`가 ✅로 박혀 있어. 게다가 6월 4일에 릴리스 나올 정도로 현역으로 굴러가는 프로젝트야. 핵심은 이놈이 배터리 값을 **로컬 HTTP XML 엔드포인트**로 뱉어준다는 거(`is_online` 키 같은 것도 포함). MQTT로도 쏴주고.

그래서 그림은 이렇게 됨:

- **백엔드(정찰)** = LGSTrayEx가 백그라운드에서 마우스랑 교신해서 배터리 % 읽음 → 로컬 주소로 XML 서빙
- **프론트엔드(네가 만들 부분)** = 그 로컬 주소를 주기적으로 폴링 → 파싱 → 작은 오버레이 창에 그려주고, 단축키로 껐다 켰다

네가 직접 만드는 건 "마우스랑 대화하는 어려운 부분"이 아니라 "예쁜 오버레이 + 토글 + 시간계산"이라서 난이도가 확 내려가. 정확한 포트/XML 형식은 LGSTrayEx 설치 후 `appsettings.toml`의 http 서버 섹션에서 확인하면 돼 (포트가 설정값이라 내가 단정 안 할게).

**"평균 남은 시간"은 마우스가 안 알려줘 — 네가 계산해야 함**

이 부분이 사실 G HUB도 안 해주는, 네가 직접 만들 가치가 있는 기능이야. 마우스는 보통 % 또는 전압만 보고하고 "몇 시간 남음"은 안 뱉어. 방법은 간단한 외삽이야: `(시각, 배터리%)`를 로그로 쌓아서 시간당 몇 %씩 빠지는지(방전율) 구하고, `남은시간 = 현재% ÷ 방전율`. 표본 쌓이기 전 초반엔 스펙값(60~90시간)을 기준선으로 깔아두면 돼.

**솔직한 경고 하나** — 일반적인 "항상 위(topmost)" 창은 테두리없는 전체화면(borderless) 게임 위엔 잘 떠도, **진짜 독점 전체화면(exclusive fullscreen) 게임 위엔 안 그려져.** FPS 하면서 띄울 거면 게임을 borderless로 돌리거나, 아니면 별도 모니터/세컨드 화면에 띄우는 식으로 가야 해. 이건 오버레이 만들기 전에 정하고 들어가야 삽질 안 함.
